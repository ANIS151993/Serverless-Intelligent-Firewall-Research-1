Introduction & Motivation

The rapid adoption of cloud-native and microservices architectures has fundamentally disrupted traditional perimeter-based network security models. Legacy firewalls, designed for static on-premise environments, fail to scale dynamically with ephemeral serverless workloads.

Simultaneously, modern cyber threats have grown in sophistication—DDoS attacks, port scanning, brute-force intrusions, and web-based exploits bypass static rule-based systems with increasing ease. Machine learning offers a principled solution: pattern recognition across millions of network flows to classify malicious behavior in real time.

Zero-Trust Architecture Gap
⚠ The ZTA-Serverless Disconnect

NIST SP 800-207 mandates Zero-Trust for federal systems, yet existing ZTA frameworks assume persistent workload identity. Serverless functions violate this assumption: they spin up in milliseconds, share execution environments, and lack static IP identifiers— creating a critical enforcement gap.

This research bridges the gap by embedding an AI-driven traffic classifier directly into the serverless execution pipeline, enforcing ZTA principles at the network-flow level without relying on static perimeter controls.

Key Contributions
  • Deep LSTM model achieving 98% accuracy on CICIDS2017 with all four metrics at parity—F1, Precision, Recall, Accuracy.
  • First serverless-native IDS deployment on AWS Lambda + Docker with sub-100 ms inference latency.
  • Integration framework mapping LSTM classifier outputs to ZTA policy enforcement actions.

  • Open-source codebase and reproducible Docker container for community adoption.
🔓
ZTA Principles Applied
🔐 Never Trust, Always Verify
Core ZTA tenet applied at every function invocation
Each Lambda invocation is treated as untrusted. The LSTM classifier re-validates the traffic signature before granting access, regardless of source IP or prior session state.
📋 Least Privilege Access
ML-driven dynamic policy scoping
Traffic classified as benign receives minimal necessary permissions. Anomalous flows trigger automated IAM policy restriction via AWS Security Hub integration.
📊 Assume Breach
Continuous re-authentication every flow
The system assumes any flow could be malicious. LSTM classifies 78-feature vectors from every packet batch, triggering micro-segmentation actions on anomaly detection.

Conclusion

We present the Serverless Intelligent Firewall (SIF), a production-ready AI-driven network security system achieving 98% accuracy across all metrics on the CICIDS2017 benchmark. By deploying a 3-layer LSTM classifier on AWS Lambda with Docker containerization, we demonstrate that deep learning IDS can operate at serverless scale with sub-100 ms latency. Zero-Trust Architecture principles are enforced at the flow level, eliminating the assumption of a trusted perimeter. Statistically validated (p < 0.05), our system outperforms CNN, SVM, and Decision Tree baselines while maintaining production scalability.
